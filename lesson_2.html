Principles of Cryptography

Cryptography is the science and art of securing communication, protecting data, and ensuring trust in digital systems. It involves techniques that transform readable information into a form that can only be understood by authorized individuals or systems. The main purpose of cryptography is to protect data confidentiality, integrity, authenticity, and accountability even in the presence of malicious actors or insecure environments. Modern cryptography combines principles from mathematics, computer science, and information theory to create secure communication channels and data protection mechanisms. It is the foundation of digital security used in applications such as secure messaging, online banking, e-commerce, and virtual private networks (VPNs).

Encryption

Definition: Encryption is the process of transforming readable data (plaintext) into an unreadable form (ciphertext) using an algorithm and a key.

Purpose: Ensures confidentiality so that only authorized parties can read the information.

Example: Credit card information is encrypted before being sent online using HTTPS.

Authentication

Verifies the identity of a user, device, or system before granting access.

Methods: Passwords, biometrics, certificates, or OTP codes.

Integrity

Ensures that data is not altered or corrupted during storage or transmission.

Achieved through hash functions and checksums for consistency verification.

Non-Repudiation

Prevents a sender from denying their involvement in a digital transaction.

Achieved using digital signatures and PKI (Public Key Infrastructure).

Hash Functions

Converts input data of any size into a fixed-length hash digest.

Examples: MD5, SHA-1, SHA-256, SHA-3.

Ciphers

Ciphers are algorithms used for encryption and decryption — divided into symmetric and asymmetric types.

Symmetric Encryption

  • Key: One shared key for encryption and decryption.
  • Examples: AES, DES, 3DES, RC6, IDEA.
  • Advantage: Fast and efficient.
  • Disadvantage: Key sharing can be insecure.

Asymmetric Encryption

  • Key: Uses a public and a private key.
  • Examples: RSA, DSA, ECDSA, Diffie-Hellman.
  • Advantage: Solves key distribution issues.
  • Disadvantage: Slower than symmetric encryption.

Transport vs End-to-End Encryption

Transport Encryption

Encrypts data between the client and the server (e.g., HTTPS). The server can still access the data.

End-to-End Encryption (E2EE)

Encrypts data on the sender’s device and decrypts it only on the recipient’s device (e.g., Signal, WhatsApp).

What is a Hypervisor

A Hypervisor, is software that enables virtualization by allowing multiple operating systems to run on a single physical machine. It manages and allocates hardware resources—such as CPU, memory, and storage—among virtual machines (VMs), ensuring they operate independently and securely.

Types of Hypervisor

There are two main types of hypervisors — Type 1 (Bare-metal) and Type 2 (Hosted).

Feature Type 1 — Bare-Metal (Native) Type 2 — Hosted
Architecture Runs directly on physical hardware without a host OS. Runs on top of an existing operating system.
Performance High performance, low latency. Less efficient due to host OS overhead.
Security More secure and isolated; fewer attack surfaces. Less secure due to dependency on host OS.
Use Case Data centers, cloud infrastructure, enterprise servers. Testing, education, and desktop virtualization.
Examples VMware ESXi, Microsoft Hyper-V, Xen, KVM VirtualBox, VMware Workstation, Parallels Desktop

Cloud Storage: 3 Varieties

  • Cloud Storage Applications: Provide online space for file storage and access.
  • File Synchronization: Keeps files updated across multiple devices.
  • Content Delivery Networks (CDN): Distribute content globally to reduce latency.

Cloud Deployment Models

Each cloud model defines how services are hosted, managed, and accessed.

Model Ownership Accessibility Advantages Challenges Use Cases
Public Cloud Owned by third-party providers (AWS, Azure, GCP) Available to anyone via the Internet Low cost, scalable, easy to deploy Limited control and shared resources Startups, SaaS, web applications
Private Cloud Owned by a single organization Internal access only Enhanced privacy and control Expensive setup, limited scalability Finance, government, healthcare
Hybrid Cloud Combination of public and private Controlled internal and external access Flexible, secure, and cost-efficient Complex integration and management Enterprises with mixed workloads
Community Cloud Shared among organizations with similar goals Restricted to authorized members Shared costs, collaboration, compliance Limited scalability, coordination issues Research, education, government groups

Note: Connecting networks to external clouds inherits their risks — always enforce strong access control.

Cloud Access Security Broker (CASB)

A CASB ensures secure and compliant use of cloud services by enforcing policies and visibility.

  • Monitors data movement between users and cloud apps.
  • Applies security rules like encryption and access restrictions.
  • Detects and mitigates risky behavior in cloud environments.
