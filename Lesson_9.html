Introduction to Event Logs

Event logs are the recorded events generated by systems, applications, and network devices. They form the primary evidence trail used for monitoring, troubleshooting, and forensic investigations on endpoints and servers.

Log Fundamentals: Why Logs Matter

  • Detect malicious activity: Logs show actions made by users and software.
  • Digital footprint: Every system interaction leaves a trace — timestamps, sources, event types.
  • Investigation evidence: Logs help reconstruct timelines and prove what happened.

The Power of Contextual Correlation

A single log entry is rarely decisive. Aggregating and correlating logs answers the key incident questions:

QuestionLog-based Evidence
What happened?Event type and description (e.g., file write, login)
When?Timestamps, synchronized with NTP
Where?Device name, IP address, or hostname
Who?User or service account recorded in the log
Outcome?Status codes, success/failure flags

Common Log Types

Application Logs

App-specific events, errors and operational messages.

Security / Audit Logs

Login events, privilege changes, audit trails for compliance.

System Logs

Kernel messages, boot events, hardware and OS errors.

Network & Web Logs

Connection records, firewall logs, web server access logs.

Log Formats & Standards

  • Structured: JSON, CSV — easy for machines to parse.
  • Semi-structured: Timestamps + message text — partially parseable.
  • Unstructured: Free-text logs — require parsing/normalization.
  • Best practice: adopt structured logging where possible to speed search & correlation.

Log Collection & Centralization

  • Collect logs from endpoints, servers, network devices, and cloud services.
  • Synchronize time across systems using NTP for accurate timelines.
  • Centralize logs in a SIEM or log store (Elastic, Splunk, etc.) for correlation and alerting.
  • Centralization enables fast searching, dashboards, and automated detection rules.

Windows Log File Characteristics

Windows event logs are stored in a binary format (`.evt`, `.evtx`). They must be viewed with the Event Viewer or parsed into XML/JSON for analysis. Common path: C:\Windows\System32\winevt\Logs.
